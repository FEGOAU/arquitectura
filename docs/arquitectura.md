# Documento de Arquitectura Empresarial

## Patrón utilizado: Cliente-Servidor con API REST
- Capa de presentación: cliente web
- Capa de lógica: servicios REST
- Capa de datos: base de datos relacional

## Control de versiones
- Trunk → rama `main` (código estable)
- Branches → ramas `feature/` (desarrollo paralelo)
- Tags → Releases de GitHub (versiones finales)
